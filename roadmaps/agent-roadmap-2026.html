TypeScript 适配

路线图原版使用 LangGraph (Python)，但你使用 TypeScript。Phase 2 的多步骤编排部分使用 Mastra 作为 TS 原生替代。Mastra 是 YC W25 项目，由 Gatsby 团队构建，专为 TypeScript 生态设计。Claude Agent SDK 继续使用（有 TS 绑定）。

📚 核心学习内容

  • Mastra 框架 — TypeScript 原生的 Agent 编排框架，状态图 + 中间件模式
  • 持久化与检查点 — 进程中断后恢复执行，检查点策略
  • Human-in-the-loop — 在关键决策点暂停等待人类确认
  • 并行子 Agent — 隔离上下文的子 Agent 并行执行，压缩摘要返回父 Agent
  • 上下文压缩策略 — 文件系统卸载、历史压缩、隔离上下文
  • 记忆选择 — 线程级（检查点）、用户级（Mem0）、自管理（文件系统），为什么不用向量数据库

兴趣探索适配

由于你的目标是兴趣探索 → 创业，CI 门控要求降为"有则更好"而非必须。但构建评估体系的思维方式对创业产品至关重要 — 你需要能证明产品没有退步。

📚 核心学习内容

  • 四种评估类型 — 单轮评估、轨迹评估、LLM-as-Judge、终态评估
  • 可观测性平台 — LangSmith（LangGraph 生态）/ Braintrust（框架无关 CI 门控）/ Inspect（基准级评估）
  • 黄金数据集 — 从生产失败中构建，而非合成数据，每季度增长 10-20%
  • 评估陷阱 — 基准污染 5-33% 高估、模型能检测是否被评估、pass^k 比 pass@1 更有意义
  • CI 门控 — PR 触发全量评估，黄金集通过率下降 ≥3 分则阻断合并

创业适配

由于你的目标是创业/做产品，Phase 5 提升为 DEEP 级别。额外增加「成本纪律」作为独立交付物 — 对创业产品而言，成本控制是生死线。

📚 核心学习内容

  • 成本纪律 — Prompt 缓存（最高 90% 节省）、按难度路由模型（Haiku/Sonnet/Opus）、Batch API 50% 折扣、多 Agent 约 15× token 消耗
  • 延迟优化 — 并行工具调用、流式输出、子 Agent 扇出（60 步串行 → 10 步主 + 5 个并行 10 步子 Agent）
  • 安全与沙箱 — Modal/E2B 代码执行沙箱、Composio 凭证代理、PreToolUse Hooks 阻断危险操作
  • 监控与漂移 — 追踪采样策略、token 成本/请求、工具失败率、LLM-as-Judge 均分、p95 延迟
  • 韧性 — 持久化执行非可选（>60s 的 Agent）、检查点、回退、分叉
  • 产品化思维 — 从技术项目到可售卖产品的转化：定价模型、用户引导、错误体验
