## Context

当前项目是一个纯前端的个人作品集网站，使用原生 HTML/CSS/JavaScript 构建。摄影集作为一个新功能模块加入，需要与现有设计风格保持一致，同时保证简约的摄影展示体验。

### 约束条件
- 纯前端实现，无后端服务
- 图片存储使用本地文件或外部图床链接
- 配置通过 JSON 文件管理，需手动编辑
- 兼容现有项目的技术栈与构建方式
- 兼容 GitHub Pages 静态托管（使用相对路径）
- 设计风格与主站统一：深色主题、渐变球体背景、颗粒纹理效果

## Goals / Non-Goals

**Goals:**
- 提供简约、沉浸式的摄影作品浏览体验
- 实现灵活的标签分类与筛选功能
- 支持主题定制，方便其他摄影爱好者复用
- 响应式设计，适配移动端与桌面端
- 图片懒加载优化性能

**Non-Goals:**
- 不实现图片上传后台界面（通过本地文件管理）
- 不实现用户认证与权限管理
- 不实现图片在线编辑功能
- 不实现评论/互动功能

## Decisions

### 1. 设计风格选择
**决定**: 采用与主站一致的深色主题设计

**设计元素**:
- 深色背景 (#0a0a0f)
- 渐变球体装饰（orb-1/2/3）
- 颗粒纹理覆盖层（grain-overlay）
- 渐变色系 (#ff6b6b, #feca57, #48dbfb, #ff9ff3)
- 毛玻璃效果卡片
- 柔和阴影与动画

**理由**:
- 与主站视觉风格统一，形成连贯的品牌体验
- 深色背景更适合摄影作品展示，减少视觉干扰
- 渐变与纹理增加视觉层次，避免单调

### 2. 技术栈选择
**决定**: 继续使用原生 HTML/CSS/JavaScript，不引入框架

**理由**:
- 与现有项目保持一致
- 功能复杂度可控，无需框架 overhead
- 便于其他用户理解和修改
- 加载速度更快，符合摄影网站需求

**备选方案**: React/Vue - 对于当前需求过于沉重

### 3. GitHub Pages 兼容性
**决定**: 使用相对路径，确保静态托管兼容

**路径规范**:
- `data/photos.json` - 照片数据
- `theme.config.json` - 主题配置
- `css/gallery.css` - 样式文件
- `js/config.js` - 配置模块
- `js/gallery.js` - 主逻辑

**理由**:
- GitHub Pages 完全支持静态文件访问
- 相对路径确保在任何部署路径下都能正常工作
- 不需要服务器端支持

### 4. 图片数据结构
**决定**: 使用单一 photos.json 文件存储所有图片元数据

```json
{
  "photos": [
    {
      "id": "unique-id",
      "title": "作品标题",
      "url": "/assets/photos/xxx.jpg",
      "thumbnail": "/assets/photos/thumbs/xxx.jpg",
      "tags": ["街拍", "人文", "黑白"],
      "date": "2024-01-15",
      "camera": "Sony A7IV",
      "settings": "f/1.8, 1/500s, ISO 400",
      "description": "作品描述（可选）"
    }
  ]
}
```

**理由**:
- 简单直观，易于手动编辑
- 单一数据源便于维护
- 支持多标签、EXIF 信息扩展

**备选方案**: 按目录分类 - 不够灵活，不支持多标签

### 3. 主题配置方案
**决定**: CSS 变量 + JSON 配置文件

```json
{
  "site": {
    "title": "Felix's Photography",
    "logo": "/assets/logo.svg",
    "favicon": "/assets/favicon.ico"
  },
  "theme": {
    "colors": {
      "primary": "#1a1a1a",
      "accent": "#ff6b35",
      "background": "#ffffff",
      "text": "#1a1a1a"
    },
    "fonts": {
      "heading": "Playfair Display",
      "body": "Noto Sans SC"
    }
  },
  "navigation": {
    "links": [
      { "label": "作品集", "href": "/gallery/" },
      { "label": "关于", "href": "/about/" }
    ]
  }
}
```

**理由**:
- CSS 变量运行时切换主题高效
- JSON 配置文件易于理解和修改
- 分离配置与代码，便于复用

**备选方案**: SASS 变量 - 需要重新编译，不够灵活

### 4. 布局方案
**决定**: 响应式瀑布流/网格混合布局

- 桌面端: 自适应三列网格
- 平板端: 两列网格
- 移动端: 单列列表
- 支持切换网格视图 / 时间线视图

**理由**:
- 网格视图适合浏览大量作品
- 时间线视图适合按时间叙事
- 响应式适配各种设备

### 5. 图片灯箱实现
**决定**: 自定义轻量级灯箱组件

功能:
- 点击图片进入灯箱
- 左右键/滑动切换图片
- ESC 键关闭
- 显示图片信息（标题、标签、拍摄参数）
- 支持键盘快捷键

**理由**:
- 控制体积，避免引入重型库
- 可定制性更强，符合设计风格
- 学习成本低，便于维护

## Risks / Trade-offs

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 手动编辑 JSON 配置容易出错 | 中 | 提供 JSON Schema 校验 + 示例配置 |
| 大量图片加载性能问题 | 高 | 实现懒加载 + 缩略图 + 图片压缩建议 |
| 无后台管理界面使用不便 | 中 | 提供详细的配置说明文档，后续可考虑本地管理工具 |
| 主题配置可能不够灵活 | 低 | 从核心配置开始，根据反馈逐步扩展 |
